信息收集

1.端口扫描

发现只有一个22端口

尝试访问一下

2.目录扫描

发现有一个zip 尝试去解压读取

3.读取zip文件

显示 Gzip compressed data：说明它是 .gz 格式，应该用 tar -xzvf backup.zip 或 gunzip

现在查看目录发现是一个backup后面没有任何后缀

再次查看

如果显示 POSIX tar archive：说明它是一个打包文件。

  • 操作：执行 tar -xvf backup 来进一步释放内容。

如果显示 ASCII text：说明它就是一个纯文本。

  • 操作：执行 cat backup 直接看内容。

如果显示 directory：说明解压直接出了个文件夹。

  • 操作：执行 cd backup 进去看看。

这里显示的是第一种那么我们直接执行

SSH 对私钥的权限要求非常严格。如果权限太开放（比如 644），SSH 会出于安全考虑拒绝使用它。所以必须将其设置为只有所有者可读

查看后发现是一个私钥 将最后一行进行base64解密后就可以看到用户名 最后ssh连接就好了

提权处理

利用sudo -l可以看到一个reboot

再看眼可读文件 看看有没有你能修改的系统服务文件

可以看到是一个sshl服务

sslh 是一个协议分流器（Protocol Multiplexer） 。它的核心功能是让多个不同的网络服务（如 HTTPS、SSH、OpenVPN 等）共享同一个监听端口（通常是 443 或 22）

接着核心思路是编写一个恶意的动态链接库（.so 文件），并将其路径写入 sslh 的环境配置文件中，最后通过重启触发 root 权限

创建一个root.c

再编译为动态链接库 同时劫持环境变量 触发重启并提权

等系统重启完后等待一会再连接一遍
