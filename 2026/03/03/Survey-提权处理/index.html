二、提权处理

先用ssh连接

再用sudo -l查看权限

查看一下内容

接着再进/opt/maze_admin/libs这个目录里查看一下

找到一个report_formatter.py文件

这段代码是一个非常典型的具有**本地提权（Privilege Escalation）**隐患的 Python 脚本。它的核心问题在于对外部库加载路径的配置不当

这段代码存在一个致命的安全缺陷：sys.path 注入。

  • sys.path.insert(0, "/opt/maze_admin/libs")： 这是漏洞的根源。Python 的 sys.path 是一个列表，决定了模块的搜索顺序。insert(0, ...) 将该目录插入到搜索路径的最顶端。这意味着 Python 会优先从这个目录查找模块，甚至早于系统标准库。
  • import report_formatter： 程序尝试加载一个名为 report_formatter 的自定义模块。
  • report_formatter.print_header()： 程序调用了该模块中的一个函数。如果这个函数被我们篡改，程序就会在不知情的情况下执行我们的恶意指令。

由于当前用户 x3x）对 /opt/maze_admin/libs 这个目录拥有写权限 所以我们就可以利用此进行提权处理

首先将原有的report_formatter.py文件删除

再构造一个恶意脚本文件来触发

确保这个文件与刚刚我们删除的文件在同一目录下

接着用whoami就可以看到是root权限了
